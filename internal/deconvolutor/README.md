# Deconvolutor reports

Plan-critique reports produced by the plan-deconvolutor pass. Each report takes a proposed
track or plan and critiques its feasibility, dependencies, and load-bearing gaps before the
work is authorized. They are kept for the record — a track that was critiqued and set aside
here may be revisited later, and these reports are why it was set aside.

## Naming convention

Every file is prefixed with the datetime it was produced, so the folder sorts chronologically
and the most recent report is always last:

```
{YYYY-MM-DD}_{HHMM}_{DESCRIPTIVE_NAME}.md
```

e.g. `2026-07-10_1901_DECONVOLUTOR_SUMMARY_both_tracks.md`. When adding a new report, use the
datetime it was written (24-hour clock). Do not renumber or rewrite existing reports.
