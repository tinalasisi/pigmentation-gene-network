## 1. The problem (quantified against the frozen data)

The module-balance metric is `(nP − nH)/(nP + nH)` over **unique genes under episodic
selection** (aBSREL corrected p<0.05) on each origin's tip branches. It counts *absolute*
selected genes per module, so it is sensitive to how many genes each module contributes to
the panel.

The current panel (`data/gene_modules.csv`, 78 genes) is **imbalanced 2:1**:

| | hormone | pigmentation |
|---|---|---|
| genes in panel | **52** | **26** |
| tested tip-branch tests | 4039 | 2031 |
| per-branch selection **rate** | 5.77% | **6.84%** |
| unique **selected genes** (metric input) | **38** | **20** |

The per-branch hit *rate* is actually **higher for pigmentation** (6.84% vs 5.77%) — i.e.
pigmentation genes are biologically *more* likely to be under selection. But because the panel
carries twice as many hormone genes, the *count* of selected genes comes out hormone-heavy
(38 vs 20), which pulls almost every origin's balance toward the hormone pole. **The current
hormone tilt is therefore substantially a panel-size artifact, not a biological signal.**

## 2. Missing pigmentation genes

Current 26-gene pigmentation panel:
`ASIP, BNC2, DCT, EDN3, EDNRB, FOXD3, IRF4, KIT, KITLG, LEF1, MC1R, MFSD12, MITF, MLANA, MRAP2, OCA2, PAX3, PMEL, POMC, SLC24A4, SLC24A5, SLC45A2, SOX10, TFAP2A, TYR, TYRP1`

The project's own curated pigmentation sources are far larger:
- **Baxter 2018** (`data/processed/baxter2018_650_pigmentation_genes.csv`) — 635 human pigmentation gene symbols.
- **Main gene network** (`data/processed/gene_network_nodes.csv`) — 168 nodes.

**Tier-1 candidates** (present in BOTH the 168-node network AND Baxter 635, absent from the current panel; 18 genes):
`BCL2, CDC42, CDH2, CTNNB1, EDN1, EGFR, HGF, HRAS, MAP2K1, MAP2K2, MAPK3, PAK1, RAC1, RACK1, RAF1, SRC, TP53, TRAF6`

Note these are largely signaling/effector genes (RAS-MAPK, WNT/CTNNB1, KIT-pathway) — appropriate for a pigmentation module, but the specialist should confirm module assignment (some are broadly pleiotropic; decide whether "pigmentation module" means melanocyte-specific or includes upstream signaling). A defensible target is **≈52 pigmentation genes to match the 52 hormone genes (1:1)**, drawn from Baxter with a documented inclusion rule.

**Decision needed from the specialist (not from me):**
- Final pigmentation gene list + module-assignment rule (melanocyte-core only, or include signaling effectors?).
- Whether to instead **normalize the metric by panel size** (e.g. rate-based `(rateP − rateH)/(rateP + rateH)`) as a cheaper alternative to a full rerun — this would fix the artifact analytically using the *existing* `branch_rates.csv` without new HyPhy jobs. **Worth evaluating first: it may resolve the issue without cluster time.**

## 3. What a full rerun regenerates (dependency chain)

The selection inference is HPC-produced (commit 4c07317). Adding genes means re-running the ortholog→align→HyPhy pipeline for the new genes, then everything downstream:

1. **Upstream (HPC / HyPhy — MOL_EVO_SPECIALIST + cluster):** pull orthologs for the added pigmentation genes across the ~117-genome primate set, align, run aBSREL (all tip branches) and RELAX (the ≥2-tip powered origins). → regenerates `data/branch_rates.csv` and `data/per_origin_K.csv`.
2. **Panel file:** update `data/gene_modules.csv` with the added genes + module labels.
3. **Downstream (local, R+Python — regenerated by the existing code):**
   - `module_selection_analysis.qmd` → recomputes `module_balance_results.csv` (the metric per origin).
   - `figures.py` → `fig_module_balance.png`, `fig_per_lineage_genes.png`.
   - `tree_figures.R` → `fig_circular_tree_balance.png`, `fig_phylo_heatmap.png`.
   - `sister_figures.py` → `fig_sister_*.png`.
   - `opie_analog_results.csv` is trait-dynamics only (phenotype + tree) — **unaffected** by the gene panel; does not need rerunning.
