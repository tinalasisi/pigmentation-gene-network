# Dichromatism selection analysis — reproducible notebook

This directory contains a single, self-contained analysis notebook that runs the complete
comparative-genomics analysis end-to-end and reports every result and figure.

## Contents

```
analysis/
├── dichromatism_selection.ipynb   # the analysis (run this)
├── README.md                      # this file
├── data/                          # frozen inputs (see below)
│   ├── dichromatism_coding.csv        # our phenotype coding (2 levels)
│   ├── relax_results.csv              # per-gene RELAX K, p, BH-p (v3 run)
│   ├── fit_health.csv                 # per-gene QC metrics + flags
│   ├── gene_panel.csv                 # 80 genes × set membership
│   ├── hormone_axes.csv               # hormone genes × endocrine axis
│   ├── pigmentation_network_nodes.csv # curated pigmentation network
│   ├── pigmentation_network_edges.csv
│   ├── primate_species_tree.nex       # dated primate species tree (topology)
│   ├── opentree_independent_tree.nwk  # independent tree (Open Tree of Life)
│   ├── v3_certification.csv           # certified-hit table
│   ├── summary.json / tip_roster.csv / alignment_qc.csv  # run provenance
└── figures/                       # regenerated by the notebook
```

## How to run

The notebook runs entirely from the frozen files in `data/` — **no cluster access, no
network, no heavy compute required.** It uses only `pandas`, `numpy`, `scipy`, `matplotlib`,
and `networkx` (the phylogeny is drawn with a small built-in newick renderer, so no
Biopython/ete3/dendropy dependency).

```bash
# from this directory
jupyter notebook dichromatism_selection.ipynb    # then Run All
# or headless:
jupyter nbconvert --to notebook --execute --inplace dichromatism_selection.ipynb
```

Every cell reads relative paths under `data/`, so the notebook is portable — clone the repo
and run.

## The two-level dichromatism coding

`dichromatism_coding.csv` is **our** phenotype assignment (it can be cross-validated against
published dichromatism assessments later). Columns:

- `dichromatic` (0/1) — the **primary foreground**. 1 = the 24 species we call dichromatic.
- `dichromatism_level` — `complete` (9 species) / `partial` (15) / `none` (93).

**The primary analysis uses all 24 dichromatic species** (`dichromatic == 1`). This is the
conservative binary: species where the source assessment was ambiguous (a speculative
"partial" not confirmed by the primary dichromatism flag) are coded `none` and excluded from
the foreground. To run the **complete-only sensitivity analysis**, filter the coding to
`dichromatism_level == "complete"` and re-run the selection step (see below).

## Regenerating the frozen inputs from scratch (optional)

The frozen CSVs are the committed outputs of the upstream pipeline. To reproduce them:

1. **Species tree (independent source).** Rather than the bundled `primate_species_tree.nex`,
   you can obtain an independent primate topology from the Open Tree of Life
   (`opentree_independent_tree.nwk` was pulled from their v3 `induced_subtree` API for the
   analysis species). Only the *topology* is used — the selection methods re-estimate branch
   lengths, so a dated vs. undated tree gives the same result. The two trees agree on all
   four dichromatism-family clades (verified separately).
2. **Genomes → CDS → alignment → RELAX.** The upstream pipeline (in `../scripts/`) downloads
   primate genome assemblies, extracts per-gene CDS (annotated genomes directly; unannotated
   via miniprot protein→genome), codon-aligns with MAFFT, and runs HyPhy RELAX per gene with
   the dichromatic tips as the `{Test}` foreground. This step needs an HPC cluster; see
   `../scripts/BUNDLE_README.md`.
3. **QC + certification.** The notebook's QC gate (gap %, K bounds, foreground size, tree
   length, alignment/reference ratio) is applied here and is fully reproduced by re-running
   the notebook against a fresh `relax_results.csv` / `fit_health.csv`.
